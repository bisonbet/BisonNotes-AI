Release Overview

Version 2.2 is a major Mac, recording-reliability, web-import, and accessibility release. The Mac app has moved from Mac Catalyst to a native macOS target while keeping the same app identity and data location. iPhone, iPad, and Apple Watch continue to use the familiar BisonNotes workflow.

Native on Mac

Mac-native windows, Settings, menus, keyboard commands, export, sharing, archive access, microphone selection, and local AI options.

Recording safeguards

The Mac recorder verifies that microphone audio actually starts, watches for capture stalls, and can save one usable track if the other fails.

Import From Link

Import supported audio, video, transcript documents, and public YouTube captions from a web address.

Accessible workflows

Clearer VoiceOver labels, Voice Control names, non-color state cues, reduced motion, and an adjustable playback scrubber.

Durable iCloud deletion

Summary-only deletions queue removal of live and backup CloudKit records if iCloud is temporarily unavailable.

Clean local-AI output

Thinking-capable MLX models keep reasoning internal; summaries and extracted actions show only the final response.

v2.2 also improves iPad navigation, playback controls, state contrast, deterministic accessibility-test startup, native Mac archive presentation, and Settings navigation.

The Native Mac App

Your existing Mac library is preserved. The native app uses the same bundle identity, app container, Core Data store, and iCloud container as the previous Catalyst version. Installing the matching v2.2 release upgrades BisonNotes instead of creating a separate empty library.

A more natural Mac workspace

  • Summaries, transcripts, players, library views, location details, and processing views can open in movable and resizable windows where appropriate.
  • Settings opens in a dedicated Mac Settings window.
  • Standard File and Edit menus expose import, export, navigation, and editing commands, with keyboard shortcuts where available.
  • Native RTF and PDF export use the Mac save workflow, and AppKit sharing sends content to compatible Mac apps and services.
  • Archive locations use security-scoped bookmarks so BisonNotes can restore access after relaunch.

Local AI on Apple silicon

MLX Swift remains the default local summarization engine on supported hardware. Native Mac users with at least 16 GB RAM can also select Ternary Bonsai 27B, an approximately 8.5 GB local model intended for laptop-class processing. It is not offered on iPhone, iPad, or the legacy Catalyst target.

iCloud note: Mac, iPhone, and iPad must use the same Apple ID and CloudKit environment. Local Debug builds use CloudKit development; TestFlight and App Store builds use production, so records do not cross between those environments.

If you delete only a summary while keeping its recording and transcript, BisonNotes now queues removal of the summary’s live record, backup record, and content-index reference. When iCloud is temporarily unavailable, that removal is retried later.

Recording Reliability on Mac

The v2.2 Mac recorder is designed to fail safely when a microphone, USB interface, Bluetooth device, or system-audio source changes unexpectedly.

  • Selectable input: Choose the built-in microphone or an available USB/Bluetooth input in Transcription Settings.
  • Confirmed start: BisonNotes does not treat recording as healthy until the first microphone buffer has been written successfully.
  • Health monitoring: A watchdog detects a missing first buffer, later capture stalls, and write failures.
  • Device recovery: If the active input changes or disconnects, completed PCM segments are preserved while BisonNotes attempts a bounded reconnection or falls back to the system default.
  • Independent track checks: Microphone and system audio are validated separately. If only one track is usable, BisonNotes saves that track instead of rejecting the whole recording.
  • Recoverable source media: If final export still fails, source segments are retained in Application Support and the recovery session is included in exported diagnostic logs.

Optional meeting audio

Enable Record Meeting Audio in Recording settings to capture audio playing from other Mac apps along with your microphone. macOS requires Screen & System Audio Recording permission. BisonNotes captures audio only; it does not save screen video.

Always confirm the live recording state before leaving a meeting unattended. Hardware, permissions, cables, Bluetooth connections, and conferencing apps remain outside BisonNotes’ control. For an important first recording on new hardware, make a short test, stop it, and play it back.

Import From Link

Choose Import From Link on the Recordings screen, or File > Import From Link… on Mac.

  • Audio and video: Direct HTTPS links can import supported M4A, MP3, WAV, CAF, AIFF/AIF, MP4, MOV, M4V, AVI, and MKV files. Video audio is extracted to M4A.
  • Transcript documents: Direct links can import TXT, MD, VTT, SRT, PDF, DOC, and DOCX documents. A transcript can be summarized without an audio file.
  • YouTube captions: Public YouTube links can be checked for caption tracks. BisonNotes does not download YouTube audio or video.
  • Pasted-transcript fallback: If YouTube blocks the caption request, BisonNotes can open the video and guide you through copying and pasting its transcript.
  • Safer downloads: Imports use HTTPS or an explicitly allowed local/private address, enforce media/document size limits, validate redirects and downloaded content, preserve useful server filenames, and clean up rejected temporary files.

Accessibility

Version 2.2 expands accessibility semantics and automated audit coverage across setup, recording, imports, recordings, playback, transcripts, summaries, settings, and Apple Watch recording.

  • VoiceOver: Main controls, content rows, playback actions, setup choices, settings, transcript/summary actions, and Watch recording states expose clearer labels, values, and hints.
  • Voice Control: Repeated actions include the associated recording, transcript, or summary title when useful, making them easier to target by name.
  • Adjustable scrubber: The audio scrubber announces elapsed and remaining time and supports 15-second accessibility seek increments.
  • Non-color status: Availability, local-only state, transcript/summary presence, recording state, transfer progress, low battery, and error recovery are expressed with text or accessibility values instead of color alone.
  • Reduce Motion: Recording indicators retain their meaning without relying on pulsing animation.
  • Keyboard access: iPad and Mac users can use standard keyboard navigation and Full Keyboard Access across common SwiftUI controls; Mac-native windows and menus should also be included in release testing.

Upgrade Checklist

  1. Let current work finish. Complete active recordings, transcriptions, summaries, and iCloud jobs before installing the update.
  2. Keep a copy of critical audio. Export any irreplaceable meeting recording before a major app or operating-system update.
  3. Install v2.2 normally. On Mac, install the new build over the existing BisonNotes app; do not delete the old app’s container or support files.
  4. Open the library first. Confirm that expected recordings, transcripts, summaries, settings, and Keep on This Device states are present.
  5. Review Mac permissions. Confirm Microphone access. If you use meeting-audio capture, also confirm Screen & System Audio Recording access in macOS System Settings.
  6. Run a short recording test. Record speech, optionally play system audio, stop, play back the result, and verify the duration before relying on the setup for a long meeting.
  7. Check selected engines. Confirm the chosen transcription and summarization engines, especially after downloading or changing a local model.

Troubleshooting and Help

A Mac recording has no microphone audio

  • Check macOS System Settings > Privacy & Security > Microphone.
  • Open BisonNotes Transcription Settings and confirm the intended input device.
  • If a USB or Bluetooth device disconnected, reconnect it and make a short test recording.
  • Export diagnostic logs before clearing settings or reinstalling; the log report lists retained recording-recovery sessions.

Meeting audio is missing but the microphone was saved

Confirm Screen & System Audio Recording permission and that the other app was producing sustained audible output. If the system track is silent or unavailable, v2.2 intentionally finalizes the usable microphone recording instead of discarding it.

A web import is rejected

Confirm the address points directly to a supported file over HTTPS, is within the supported size limit, and does not require an interactive login. Signed download links are supported when the final response is valid media or a supported transcript document.
